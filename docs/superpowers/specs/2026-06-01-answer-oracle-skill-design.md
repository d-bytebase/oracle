# twinkle ✨ — 答案机 / Answer Oracle Skills 设计稿

日期:2026-06-01
状态:已与用户对齐,待实现

## 一句话

两个 fortune-cookie 风格的 Claude Skill,随机抽一句**童趣、鼓励、有趣**的话当"今日答案"。
中文用 `/答案`,英文用 `/lucky`。灵感来自"答案之书 / 答案机",但**成品与任何品牌无关**。

## 目标

- 一个轻量、好玩、随手可用的"许愿/答案"小工具。
- 抽签式(fortune cookie):答案是预写好的池子,随机抽一条,**与用户问什么无关**。
- 双语:`/答案`(中文)、`/lucky`(English),各自独立的答案库。
- 真随机,不靠 Claude"自己挑"(避免偏向/重样)。

## 非目标(Non-goals)

- **不绑定任何品牌**:无麦当劳、无薯条/蘸酱/可乐、无任何食物或商业元素。
- **不读题作答**:不是 Claude 理解问题再生成答案;问题只是仪式,抽签与问题内容无关。
- 不做 MCP / 服务端;纯 Skill 形态。
- 不做联网、不做持久化统计(抽中记录等),先不要(YAGNI)。

## 形态

两个 **personal skill**,装在 `~/.claude/skills/`,任何项目里都能用:

```
~/.claude/skills/
  答案/    SKILL.md + answers.txt   (中文,500 行)
  lucky/   SKILL.md + answers.txt   (English, 500 lines)
```

源文件 + 生成脚本放在本仓库 `twinkle/` 里做版本管理;装好后同步到上面的 skills 目录。

### ⚠️ 实现第一步必须先验证

`/答案` 这种**中文名 slash command** Claude Code 是否支持(skill 名 / 命令名能否用非 ASCII)。
- 若支持:命令就用 `/答案`。
- 若不支持:回退到拼音 `/dayan` 或英文别名,但优先想办法保住 `/答案` 的体验。
这一步在写任何答案库之前先做,避免白做。

## 声音 / 内容

- 调性:**童趣 + 鼓励为主**,温柔、有趣、普世治愈,一点点玄学留白。
- 数量:中英文**各 500 条**。
- 为避免 500 条稀释/重样,按**系列**组织(抽签时全池随机,系列只是写作时的多样性保证):
  - 勇气「去做吧」
  - 治愈「没关系」
  - 童心「今天当个小孩」
  - 时机「再等等」(玄学留白)
  - 玩闹 / 惊喜
- 英文库镜像同样的系列,但是**原创英文**,不是中文的直译。

中文示例:
- 「今天不用长大,你已经很勇敢了。」
- 「是的,去做吧——最坏的结果也没你想的那么坏。」
- 「再等等,好东西正在路上,它只是迷路了一下下。」

English 示例:
- "Yes. Go do the brave little thing."
- "It's okay to not be okay today. Tomorrow's still yours."
- "The answer is closer than you think. Turn around."

## 抽签机制(关键)

纯 Skill,但要**真随机**:SKILL.md 指示 Claude 跑一行 shell 抽一条,而不是自己挑。

- 首选:`shuf -n 1 answers.txt`
- 回退(无 `shuf`,如某些 macOS 默认环境):`awk` / `$RANDOM` 取随机行,例如
  `awk 'BEGIN{srand()} {a[NR]=$0} END{print a[int(rand()*NR)+1]}' answers.txt`

这样避免 Claude 自己挑导致的偏向、重样。

## 输出 / 仪式感

- 支持可选问题参数:`/答案 我该不该辞职`。问题只是仪式,**不影响抽签**;输出时回显"你问 → 答案"增加仪式感。
- 用水晶球 🔮 开场 + 小卡片框住答案:

```
🔮 你摇了摇水晶球…

   ┌──────────────────────┐
   │ 今天不用长大,         │
   │ 你已经很勇敢了。       │
   └──────────────────────┘
```

英文版同理:`✨/🔮 ... a little card ...`(以 🔮 为主)。

## 内容生成方案

1000 条手写不现实,用 workflow 按**系列分批 fan-out** 生成 + **去重**:
- 每个系列一个 agent,产出该系列的 N 条(中/英分开)。
- 汇总后做一次去重 + 质量过滤(去掉雷同句式、过长、跑题的)。
- 不足 500 的系列补抽,直到各满 500。
- 全程保证:无品牌/食物词、调性统一、长度适中(适合卡片)。

## 成功标准

- 在任意目录输入 `/答案`(或验证后的回退名)和 `/lucky` 能抽到一条答案,带 🔮 卡片。
- 连续抽多次明显不重样(真随机生效)。
- 答案库各 500 条,无品牌/食物词,调性统一为童趣+鼓励。
- `/答案 <问题>` 能回显问题但答案与问题无关。

## 测试

- 跑抽签命令 N 次(如 20 次),确认无报错、无重复偏向、卡片渲染正常。
- 在没有 `shuf` 的环境验证回退分支。
- 抽样人工读 ~30 条,确认无品牌/食物词、无跑题、长度适合卡片。
